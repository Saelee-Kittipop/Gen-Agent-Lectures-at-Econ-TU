Introduction to Generative Agents

Generative Agents คืออะไร?

Generative Agents คือ ตัวแทน AI (Software Agent) ที่สามารถ
จำ ประสบการณ์ที่ผ่านมา, คิดวิเคราะห์ จากความทรงจำ,
และ ตัดสินใจ ทำสิ่งต่างๆ เหมือนมนุษย์จริงๆ

แทนที่จะเขียนกฎตายตัว (if-else) ให้ AI ทำตาม
เราให้ LLM (Large Language Model) เป็นคนคิดและตัดสินใจเอง

เหมือนกับให้ตัวละครในเกม "มีจิตใจ" ของตัวเอง
ไม่ใช่แค่เดินตาม script ที่เขียนไว้ล่วงหน้า

อธิบายแบบง่ายๆ - เปรียบเหมือน NPC ในเกมที่มีหัวคิดเป็นของตัวเอง

3 ประเภทของ Memory

Observation สิ่งที่เห็น/สังเกตโดยตรง
เช่น "เห็นป้ายลดราคานม 30% ที่ Big C"

Reflection ข้อสรุปที่คิดได้จากหลายๆ ความทรงจำ
เช่น "Big C มักมีโปรนมดีๆ ควรไปเช็คบ่อยๆ"

Plan แผนที่ตั้งใจจะทำ
เช่น "วันเสาร์นี้จะไป Big C ซื้อนมตอนเช้า"

Memory Retrieval Formula

เวลาจะดึงความทรงจำ ใช้สูตรนี้คำนวณคะแนน

score = recency × importance × relevance

Recency (ความใหม่)

recency = 0.99 ^ hours

ความทรงจำเก่าจะได้คะแนนน้อยลงตามเวลา
(Exponential Decay)

  • 1 ชม. → 0.99
  • 24 ชม. → 0.79
  • 72 ชม. → 0.49
  • 168 ชม. (1 สัปดาห์) → 0.18

Importance (ความสำคัญ)

importance = 1-10

LLM ให้คะแนนว่าเรื่องนี้สำคัญแค่ไหน

  • 1 = เรื่องทั่วไป (เห็นคนเดินผ่าน)
  • 5 = สำคัญปานกลาง (เห็นโปรลด 20%)
  • 10 = สำคัญมาก (ตกงาน, แต่งงาน)

Relevance (ความเกี่ยวข้อง)

relevance = cosine_similarity

ความทรงจำนี้เกี่ยวกับคำถามปัจจุบันแค่ไหน
(0 = ไม่เกี่ยว, 1 = ตรงเป๊ะ)

Recency Decay Graph

recency = 0.99 ^ hours (ยิ่งนานยิ่งลืม)

แกน X = จำนวนชั่วโมงที่ผ่านไป, แกน Y = คะแนน Recency

สังเกต: หลัง 72 ชั่วโมง (3 วัน) คะแนนเหลือแค่ ~0.49
หลัง 1 สัปดาห์ เหลือแค่ ~0.18 — Agent จะลืมเรื่องเก่าๆ ไปตามธรรมชาติ

Memory Retrieval ในโค้ดจริง

class MemoryStream:
    RECENCY_DECAY = 0.99  # Decay factor per hour

    def retrieve(self, query, top_k=5):
        """Retrieve top-k most relevant memories"""
        scored_memories = []

        for memory in self.memories:
            recency = self.RECENCY_DECAY ** hours_elapsed
            importance = memory.importance / 10.0
            relevance = self._calculate_relevance(memory, query)

            # Combined score
            score = recency * importance * relevance
            scored_memories.append((memory, score))

        scored_memories.sort(key=lambda x: x[1], reverse=True)
        return scored_memories[:top_k]

Reflection Mechanism

กลไกการสะท้อนคิด

Reflection คืออะไร?

Reflection = การที่ Agent สร้าง ข้อสรุประดับสูง (Higher-level Insight)
จากหลายๆ ความทรงจำรวมกัน — เหมือนมนุษย์ที่ "ตกผลึกความคิด"

เมื่อไหร่จะ Reflect?

เมื่อ ผลรวม importance ของ Observation ใหม่ๆ เกินค่า threshold

sum(importance of recent observations) > threshold → trigger reflection

เช่น ถ้า threshold = 20 และมี observations ใหม่ 3 อัน (importance 7+8+6 = 21) → สะท้อนคิด!

ตัวอย่าง Reflection

Observations (ข้อสังเกตดิบ):

Obs "ไป Big C วันเสาร์ เห็นป้ายลดราคานม 30%"

Obs "ซื้อนม 3 กล่องในราคาถูก ประหยัดได้ 45 บาท"

Obs "ของหมดเร็วมาก ถ้ามาช้ากว่านี้อาจไม่ทัน"

Obs "พนักงานบอกว่าโปรนมมาทุกสัปดาห์แรกของเดือน"

↓ LLM สะท้อนคิด ↓

Reflection (ข้อสรุปใหม่):

Reflection "Big C มีโปรโมชั่นนมดีๆ ช่วงต้นเดือน ควรไปเช้าวันเสาร์เพราะของหมดเร็ว"

Reflection นี้จะถูกเก็บกลับเข้า Memory Stream
และสามารถถูกดึงมาใช้ในอนาคตได้ — เหมือนเราจำ "บทเรียน" ได้

Reflection ในโค้ดจริง

def reflect(self, decision, result, current_time):
    """Reflect on the experience and create new memories"""
    action = decision.get("decision", {}).get("action", "stay_home")

    if action == "go_shopping":
        experience = self._build_experience_summary(decision, result)

        # Ask LLM to reflect on the experience
        # LLM returns: { "new_memory": "...", "importance": 1-10 }
        reflection = self.llm.reflect(
            self.customer, experience)

        # Store the new reflection in Memory Stream
        self.memory.add(
            content=reflection["new_memory"],
            importance=reflection["importance"],

            related_items=items_list)

        recency = self._calculate_recency(memory, current_time)
        importance = memory.importance / 10.0  # Normalize to 0-1
        relevance = self._calculate_relevance(memory, query)

        score = recency * importance * relevance

        if score >= min_score:
            scored_memories.append((memory, score))
            memory.access_count += 1

    scored_memories.sort(key=lambda x: x[1], reverse=True)
    return scored_memories[:top_k]

นี่คือ core algorithm ของ Memory Retrieval
recency × importance × relevance = final score

3. Cognitive Loop (src/customer.py)

def step(self):
    """Execute one step of the cognitive loop"""
    current_time = self.model.current_datetime

    # 1. PERCEIVE - Observe stores, promos, inventory
    perception = self.perceive(current_time)

    # 2. RETRIEVE - Pull relevant memories
    memories, scores = self.retrieve_with_scores(perception["query"])

    # 3. PLAN - LLM decides what to do
    decision = self.plan(perception, memories)

    # 4. EXECUTE - Take action (go shopping, stay home)
    result = self.execute(decision, current_time)

    # 5. REFLECT - Create new insights from experience
    self.reflect(decision, result, current_time)

    return decision

แค่ 5 ขั้นตอน! ง่ายและสะอาด
PLAN และ REFLECT เรียก LLM, ที่เหลือเป็น rule-based
